# Testnet UX UI Polish Follow-On (March 2026)

This branch is the follow-on to the testnet UX stack and is intentionally limited to UX and UI polish.

It exists to correct the tone, composition, and usability issues introduced by the earlier testnet launch branches without extending the information architecture further.

## Scope

- tighten homepage composition instead of adding more surface area
- improve CTA clarity without making the site feel like an internal tool
- simplify testnet path presentation for storage users and provider operators
- preserve the strongest parts of the original homepage visual language

## Non-Goals

- adding more routes just to explain the product
- merging stacked branches into `main`
- expanding protocol or backend scope in this branch

## Working Principles

- fewer decisions on first view
- stronger hierarchy and more visual conviction
- less operational copy in primary surfaces
- details should move behind secondary affordances, not into the hero
